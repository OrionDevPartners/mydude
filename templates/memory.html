{# ── How it works ── #}

How It Works

  • Local KG (Cognee) — embedded knowledge graph on disk. Entities and relations are extracted from every task's facts and decisions using pattern-based NER and TF-IDF indexing.
  • Cloud store (Mem0) — cloud-backed memory layer. Runs in local-file fallback mode when MEM0_API_KEY is unset.
  • Recall — before wave 0, semantically related prior memories are injected into the swarm's fact set via TF-IDF cosine similarity (replaces keyword Jaccard).
  • Persist — after the final wave, load-bearing facts, decisions, and VERIFIED claims are written to long-term memory keyed by goal topic.
  • Semantic contradiction check — TF-IDF cosine + temporal-conflict detection. Catches "deadline is Monday" vs "finish by Friday" type contradictions that pure keyword overlap misses.
  • Sync bridge — bidirectional rsync between local and cloud. Last-writer-wins on timestamp; VERIFIED claims are never overwritten by lower-confidence entries.
  • Consolidate — promotes high-confidence memories to VERIFIED status and applies exponential decay to rarely-accessed entries.
  • RECURSIVE_REASONER role — a new cognitive role that traverses the KG across prior tasks to surface remembered facts and flag semantic contradictions with verified long-term memories.
