4) Low-latency design rules (non-negotiable)



Rule L1 — Local-first + speculative execution


•Start internal retrieval (Seg 13–20) immediately.
•In parallel, CPAA decides if external retrieval (Seg 21–30) is needed.
•If internal confidence crosses threshold early → cancel external.



Rule L2 — Macro-system parallelism


These run concurrently whenever possible:

•S2 Internal Retrieval ↔ S1 Pathway Planning
•S3 External Retrieval ↔ S4 Cross-domain bridging
•S5 Consensus ↔ S7 Validation (early sanity checks)



Rule L3 — Incremental output streaming


Even in Standard mode:

•Return partial “safe preliminary synthesis” once S2 + S5 finish.
•Append “validated/expanded” if S7 completes within budget.



Rule L4 — Cache aggressively (but safely)


•Cache evidence bundles by normalized query fingerprint.
•Cache KG traversals and vector search results.
•Never cache regulated payloads unless encrypted + egress none.



Rule L5 — Divergence-driven escalation (not curiosity-driven)


External calls happen when:

•confidence < threshold
•divergence between internal modules is high
•freshness required (CAP-E001 style)
