### The Architecture: Shadow Graph & The Kill List
To preserve \mathcal{O}(1) inference latency, Memgraph acts as the **Control Plane** (calculating who dies), while local Python memory acts as the **Data Plane** (executing the execution).
 1. **The Shadow Graph:** Memgraph holds a structural replica of your Radix Tree. Every node represents a cached sequence, storing its token length, compute cost, and a frequency/recency counter.
 2. **Asynchronous Syncing:** During active loops, MLX reads from the local UMA RAM. Every few seconds, a lightweight background Celery task pushes access statistics (via the Valkey Comm Bus) to Memgraph.
 3. **The Eviction Sweep:** When psutil detects that the macOS unified memory is approaching the redline (e.g., >85% utilization), it triggers Memgraph to run the deterministic eviction algorithm, returning a "Kill List" of cache pointers to drop.

### Mapping the Logic in Memgraph (Cypher)
Because Memgraph is natively designed for this, the algorithm is executed directly inside the graph using Cypher.
**1. The Graph Schema**
Nodes are (CacheBlock) and relationships are [:FOLLOWS].
```cypher
// Creating a branch in the Shadow Graph
CREATE (root:CacheBlock {hash: "sys_prompt_v1", tokens: 500, hits: 100, last_access: timestamp()})
CREATE (leaf1:CacheBlock {hash: "tool_search_data", tokens: 2500, hits: 2, last_access: timestamp()})
CREATE (root)-[:FOLLOWS]->(leaf1);

```
**2. The Eviction Query (The Reaper Algorithm)**
When RAM hits the threshold, this Cypher query deterministically identifies the heaviest, least valuable branches. It calculates the aggregate cost of an entire branch (a node and all its children) because you cannot evict a parent without invalidating its children.
```cypher
MATCH (n:CacheBlock)
// Find leaf nodes or branches with no current active agent locks
WHERE n.active_locks = 0 
// Calculate the deterministic score (simulating the math above)
WITH n, 
     ((n.hits * 1.5) + (n.tokens * 1.0)) / (timestamp() - n.last_access + 1) AS retention_score
ORDER BY retention_score ASC
LIMIT 50 // The Kill List size
// Return the hashes to be destroyed in Python
RETURN n.hash, retention_score, n.tokens;

```
### Execution Steps (The Final Loop)
 1. **Trigger:** psutil detects system RAM hits **85%**.
 2. **Calculate:** The orchestrator fires the "Reaper Query" to Memgraph.
 3. **Purge Graph:** Memgraph deletes the selected nodes and their [:FOLLOWS] relationships in the shadow graph.
 4. **Purge RAM:** Memgraph returns the n.hash list to Python. Python executes del session_cache_store[hash], unreferencing the MLX mx.array.
 5. **Garbage Collection:** Python's garbage collector frees the objects, and Apple's UMA instantly reclaims the physical RAM for the active agent loops.
By leveraging Memgraph to map out the heavy computational math in C++, you keep the Python orchestrator completely focused on asynchronous routing, completely eliminating OOM crashes during massive multi-agent execution.
